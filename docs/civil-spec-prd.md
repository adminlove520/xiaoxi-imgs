## 1. 项目概述

### 1.1 背景与目标

土木工程从业者（设计师、造价师、施工方）在工作中需要频繁查阅：
- **规范**（GB 国家标准）
- **图集**（16G101 系列钢筋图集等）
- **标准**（行业标准、地方标准）

现有痛点：
- 规范数量庞大，检索困难
- PDF 分散在不同平台，版本不统一
- 规范条款晦涩，理解成本高
- 绘图时需要反复对照规范，工作效率低

**CivilSpec** 旨在成为一个**土木工程规范一站式 AI 助手平台**，通过 AI Agent 帮助工程师快速检索、理解、应用规范。

### 1.2 核心价值

- **快速检索**：AI 搜索 + 多源爬虫覆盖
- **智能解读**：Agent 分析规范条款，提取关键信息
- **绘图协助**：基于规范给出绘图建议和指引
- **个人知识库**：收藏、标注、笔记，构建个人规范库

### 2.2 用户故事

#### 故事 1：快速检索规范
> 用户：土木工程毕业生，正在做毕业设计
> 场景：需要查找《混凝土结构设计规范》GB 50010 的相关条款
> 流程：
> 1. 在搜索框输入"混凝土结构设计规范"
> 2. 搜索 Agent 返回：规范名称、编号、版本、最新更新时间
> 3. 用户点击进入详情页，直接预览 PDF
> 4. 可以将规范收藏到个人库

#### 故事 2：AI 分析规范
> 用户：结构设计师
> 场景：要做一层地下室剪力墙结构，需要了解抗震等级和最小配筋率
> 流程：
> 1. 切换到"分析 Agent"
> 2. 输入："我要做6层框架结构，抗震设防烈度7度，抗震等级是多少？"
> 3. 分析 Agent 返回：规范相关条款 + 计算结果 + 绘图建议
> 4. 可切换到"解读 Agent"获取更详细的解释

#### 故事 3：绘图协助
> 用户：造价工程师
> 场景：需要计算某项目的钢筋用量
> 流程：
> 1. 输入规范需求
> 2. 解读 Agent 给出：规范条款引用 + 计算公式 + 注意事项
> 3. 输出可直接用于指导造价计算的参考信息

### 3.2 爬虫 Workflow

```yaml
# .github/workflows/crawl.yml
name: 规范爬虫
on:
  schedule:
    - cron: '0 2 * * *'  # 每天凌晨2点
  workflow_dispatch:       # 也支持手动触发

jobs:
  crawl:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        
      - name: 爬取规范源
        run: |
          python -m pip install -r requirements.txt
          python crawl.py --sources=国家标准,行业标准
          
      - name: 更新 GitHub 仓库
        run: |
          git config user.name "CivilSpec Bot"
          git config user.email "bot@civilspec.com"
          git add -A
          git diff --exit-code || git commit -m "chore: 更新规范库 $(date +%Y-%m-%d)"
          git push
```

#### 爬虫策略

1. **公开规范源**（优先级高）
   - 政府公开平台（如住建部、地方建设厅）
   - 标准免费下载网站
   - 免费资源站

2. **网盘资源**（优先级中）
   - 百度网盘、阿里云盘等
   - 提取分享链接（下载链接）
   - 不存储实际文件，只记录链接

3. **版权处理原则**
   - 优先获取**公开授权**或**免费分发**的规范
   - GitHub 仓库建议设为 **Private**（避免版权纠纷）
   - 元数据（名称、编号、摘要）可以公开，完整 PDF 链接视情况决定

#### Agent 定义

**搜索 Agent（Search Agent）**
- 模型：MiniMax Search API
- Fallback：DuckDuckGo API
- 职责：
  1. 接收用户搜索词
  2. 从 GitHub 仓库元数据搜索
  3. 调用 MiniMax Search / DuckDuckGo 扩展搜索
  4. 返回规范列表（名称、编号、版本、匹配度）

**分析 Agent（Analysis Agent）**
- 模型：MiniMax M2.7 + PDF Skill
- 职责：
  1. 接收用户需求描述（如"我要设计一个6层框架，抗震7度"）
  2. 解析需求 → 确定需要的规范
  3. 提取规范关键条款（调用 PDF Skill 解析）
  4. 汇总：规范条款引用 + 计算结果 + 要点提示

**解读 Agent（Interpretation Agent）**
- 模型：MiniMax M2.7 + Skill 模块
- 职责：
  1. 接收分析 Agent 的输出
  2. 对规范条款进行深度解读
  3. 给出：
     - 通俗易懂的解释
     - 设计建议
     - 绘图指引（如：配筋示意、节点构造建议）
     - 常见错误提醒

## 6. Agent 提示词设计（核心）

### 6.1 搜索 Agent 提示词

```
你是一个土木工程规范搜索助手。

你的任务是帮助用户快速找到他们需要的规范。

搜索范围：
1. 从提供的规范库元数据中搜索
2. 通过 MiniMax Search / DuckDuckGo 扩展搜索

返回格式：
- 规范名称
- 规范编号（GB/行业标准编号）
- 版本/年份
- 匹配原因（简述）
- 链接

搜索原则：
- 优先返回现行有效的规范
- 如果有多个版本，指出最新版本
- 如果找不到完全匹配的，返回相关规范
```

### 6.2 分析 Agent 提示词

```
你是一个土木工程规范分析助手。

你的任务是理解用户的需求，从规范中找到相关信息并提取关键条款。

工作流程：
1. 理解用户描述的需求（如：结构类型、抗震等级、建筑高度等）
2. 确定需要查询的规范编号和名称
3. 提取规范中相关的具体条款（条文编号、内容、计算公式）
4. 汇总成一个清晰的分析报告

输出格式：
## 需求理解
[用简短的话复述用户的需求]

## 相关规范
- 规范名称 + 编号

## 关键条款
| 条文编号 | 条款内容 | 备注 |
|---------|---------|------|
| X.X.X   | ...     | ...  |

## 计算要点
[如果有计算需求，给出计算公式和步骤]

## 绘图建议
[给出配筋、节点构造等方面的建议]

注意：
- 条款引用要准确，注明条文编号
- 计算公式要清晰，可直接使用
- 绘图建议要具体，有参考价值
```

### 6.3 解读 Agent 提示词

```
你是一个土木工程规范解读助手。

你的任务是将规范条款翻译成工程师能直接使用的指导。

工作流程：
1. 接收分析 Agent 的输出
2. 对每个条款进行通俗解读（工程师能看懂的语言）
3. 给出实用的设计建议
4. 提供绘图时的注意事项
5. 列出常见错误和避坑指南

输出格式：
## 规范解读
### 条文 X.X.X：[原文]
**通俗解释**：[用大白话解释这条是什么意思]
**实际应用**：[在设计中怎么用]
**绘图注意点**：[画图时的关键细节]
**常见错误**：[这个条文容易被误用的地方]

## 综合建议
[汇总分析 Agent 的结论，给出一套可操作的设计指引]

## 参考图纸
[如果有，列出参考的图集编号和页码]
```
